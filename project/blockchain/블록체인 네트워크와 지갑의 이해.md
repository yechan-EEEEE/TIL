# **🌐 블록체인 네트워크 분류와 지갑의 원리**

내일 스터디에서 바로 활용할 수 있도록, 핵심 개념을 '비유'와 '흐름' 중심으로 정리한 가이드입니다.

## **1\. 블록체인 네트워크의 분류**

블록체인은 '누가 참여할 수 있느냐'와 '누가 관리하느냐'에 따라 크게 세 가지로 나뉩니다.

| 분류 | 특징 | 비유 | 대표 사례 |
| :---- | :---- | :---- | :---- |
| **퍼블릭 (Public)** | 누구나 참여하고 데이터를 읽을 수 있는 개방형 네트워크 | **광장**: 누구나 들어와서 떠들고 구경할 수 있음 | Bitcoin, Ethereum |
| **프라이빗 (Private)** | 승인된 특정 기관이나 사람만 참여 가능한 폐쇄형 네트워크 | **기업 인트라넷**: 사번이 있는 직원만 접속 가능 | Hyperledger Fabric |
| **컨소시엄 (Consortium)** | 여러 기관이 연합하여 운영하는 중간 형태의 네트워크 | **은행 연합**: 여러 은행이 모여 공동의 망을 관리 | R3 Corda, Quorum |

### **💡 핵심 포인트**

* **퍼블릭**은 탈중앙화가 가장 강하지만 속도가 상대적으로 느릴 수 있습니다.  
* **프라이빗/컨소시엄**은 신뢰할 수 있는 사람들끼리만 하기 때문에 속도가 빠르고 기업 환경에 적합합니다.

## **2\. 지갑의 핵심: EOA vs CA**

이더리움 같은 스마트 컨트랙트 기반 블록체인에는 두 종류의 '계정(주소)'이 있습니다.

### **① EOA (Externally Owned Account, 외부 소유 계정)**

* **개념:** 우리가 흔히 말하는 '지갑 주소'입니다.  
* **특징:** 개인키(Private Key)를 가진 사람이 직접 통제합니다. 스스로 트랜잭션(송금 등)을 시작할 수 있습니다.  
* **비유:** 우리가 사용하는 **개인 은행 통장**.

### **② CA (Contract Account, 컨트랙트 주소)**

* **개념:** 스마트 컨트랙트 코드가 담긴 주소입니다.  
* **특징:** 개인키가 없습니다. 즉, 스스로 움직이지 못하고 외부(EOA)에서 신호를 주어야만 정해진 코드대로 실행됩니다.  
* **비유:** **자판기**. 돈을 넣거나 버튼을 누르는(EOA의 호출) 동작이 있어야만 음료수를 내뱉습니다.

## **3\. 지갑의 원리: 키(Key)의 흐름**

지갑을 처음 만들 때 발생하는 과정을 이해하는 것이 가장 중요합니다.

### **🔑 3단계 흐름: 니모닉 → 개인키 → 공개키 → 주소**

1. **니모닉 (Mnemonic): "최후의 보루"**  
   * 12개 또는 24개의 영어 단어 나열입니다.  
   * 복잡한 숫자와 문자로 된 '개인키'를 사람이 외우기 힘들어서 만든 **복구용 단어**입니다.  
   * *주의:* 니모닉을 뺏기면 모든 돈을 뺏기는 것과 같습니다.  
2. **개인키 (Private Key): "인감 도장"**  
   * 니모닉을 통해 수학적으로 생성됩니다.  
   * 자금을 전송할 때 '내가 주인임을 증명'하는 **디지털 서명**에 쓰입니다. 절대 남에게 알려주면 안 됩니다.  
3. **공개키 (Public Key): "계좌 번호의 원형"**  
   * 개인키로부터 만들어지지만, 거꾸로 공개키를 안다고 해서 개인키를 알아낼 수는 없습니다 (일방향성).  
4. **지갑 주소 (Address): "실제 계좌 번호"**  
   * 공개키를 읽기 좋게 변환한 최종 형태입니다. 돈을 받을 때 상대방에게 알려주는 정보입니다.

## **👨‍🏫 스터디 답변 꿀팁 (비유로 설명하기)**

**질문: "개인키랑 주소가 어떻게 다른가요?"**

**답변:** "개인키는 **인감도장**이고, 지갑 주소는 **계좌번호**라고 생각하시면 돼요. 계좌번호(주소)는 누구에게나 알려줘도 되지만, 인감도장(개인키)은 내가 돈을 보낼 때 찍어야 하는 것이라 절대 보여주면 안 됩니다. 그리고 이 모든 도장과 계좌번호를 잃어버렸을 때 다시 찾아올 수 있는 마법의 주문이 바로 **니모닉**입니다."

## **📝 요약 체크리스트**

* \[ \] 퍼블릭은 '모두', 프라이빗은 '승인된 소수'가 쓴다.  
* \[ \] EOA는 사람이 쓰는 지갑, CA는 프로그램이 돌아가는 주소다.  
* \[ \] 니모닉은 개인키를 만들기 위한 단어 뭉치다.  
* \[ \] 개인키(도장) \-\> 공개키 \-\> 주소(계좌번호) 순서로 생성된다.
