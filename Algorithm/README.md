# 알고리즘
- 유한한 단계로 문제를 해결하기 위한 절차, 방법
## 알고리즘 표현 방법
- 의사코드(슈도코드, Pseudocode)
- 순서도
- 프로그래밍 언어
## 좋은 알고리즘이란?
- 정확성: 정확한 동작
- 작업량: 적은 연산으로 결과 도출
- 메모리 사용량: 적은 메모리 사용
- 단순성: 단순한 알고리즘
- 최적성: 개선할 여지 없이 최적화
# 복잡도
## 시간 복잡도
- 실제 걸리는 시간 측정
- 실행되는 명령문 개수 계산
## 시간 복잡도 ≒ 빅-오(O)표기법
- 알고리즘이 가장 느리게 실행될 때의 성능
- 시간 복잡도 함수 중 가장 큰 영향력을 주는 n에 대할 항만? 을 표시
- 계수(Coefficient)는 생략
- ![](image/시간복잡도1.jpg)
- ![](image/시간복잡도2.jpg)
- 실행시간 시간복잡도 logN,N,NlogN 안쪽으로 가능하게 짜기(난이도가 높고 확신이 없을 때 시간복잡도부터)
- 시간복잡도 : 연산의 작업량, 수행시간
    - 최선의 경우 (Bset Case): 빅 오메가 표현법
    - 평균적인 경우 (Average Case): 빅 세타 표기법
    - 최악의 경우 (Worst Case): 빅 오 표기법
- 공간 복잡도: 메모리 사용량
