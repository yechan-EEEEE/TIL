📚 데이터 파이프라인과 WSL(Windows Subsystem for Linux) 완벽 정리

이 문서는 데이터 파이프라인의 개념부터 리눅스 운영체제, 그리고 윈도우에서 리눅스를 사용하는 WSL 환경 구축까지의 내용을 담고 있습니다.

1. 데이터의 시대와 데이터 파이프라인

🔹 정보(Information) vs 데이터(Data)

정보 (1970~2000년대): 과거에는 정의되고 유의미하게 도움되는 '정보'가 핵심이었습니다.

데이터 (2010년대 이후): 현재는 대규모로 수집된 '원시 자료(Raw Data)' 자체의 중요성이 강조됩니다. 나중에 의미를 부여하여 가치를 창출합니다.

🔹 데이터 파이프라인이란?

데이터를 추출(Extract), 정제(Transform)하여 저장(Load)하고, 이후 분석 및 시각화하는 일련의 자동화된 흐름을 말합니다.

"좋은 모델은 좋은 파이프라인에서 나온다." (데이터 엔지니어의 핵심 역할)

2. 데이터 파이프라인의 핵심 개념 (ETL vs ELT)

데이터를 어떻게 처리하고 저장하느냐에 따라 두 가지 방식으로 나뉩니다.

항목

ETL (Extract, Transform, Load)

ELT (Extract, Load, Transform)

순서

추출 → 가공 → 저장

추출 → 저장 → 가공

특징

전통적인 방식, 저장소 공간 절약

클라우드 시대에 적합, 유연성 높음

스키마

Schema-on-Write (저장 시점에 결정)

Schema-on-Read (읽는 시점에 결정)

🔹 데이터 처리 방식

배치 처리 (Batch Processing): 데이터를 일정 기간 모아서 한 번에 처리 (예: 하루 1회 정산).

스트리밍 처리 (Stream Processing): 데이터가 생성되는 즉시 실시간으로 처리 (예: 실시간 로그 분석).

3. 데이터 저장소의 종류

데이터 웨어하우스 (Data Warehouse): 정형 데이터를 분석하기 위해 구조화하여 저장하는 곳. (비즈니스 의사결정 중심)

데이터 레이크 (Data Lake): 정형, 비정형(이미지, 오디오 등) 데이터를 원형 그대로 대규모로 저장하는 곳.

데이터 레이크하우스 (Data Lakehouse): 데이터 레이크의 유연성과 웨어하우스의 관리 기능을 결합한 최신 구조.

4. 리눅스(Linux)와 WSL

🔹 리눅스란?

유닉스(UNIX) 기반으로 만들어진 오픈소스 운영체제입니다.

장점: 무료, 가볍고 빠름, 전 세계 서버 시장 점유율 1위, 개발 환경(Docker, Git 등)과의 높은 호환성.

🔹 WSL (Windows Subsystem for Linux)

윈도우 환경 내에서 별도의 가상 머신 설치 없이 리눅스를 실행할 수 있게 해주는 도구입니다.

WSL 1 vs WSL 2: WSL 2는 실제 리눅스 커널을 포함하고 있어 성능이 더 뛰어나며 Docker 사용이 가능합니다.

5. 리눅스 기본 명령어 가이드

리눅스는 CLI(Command Line Interface) 기반으로 명령어를 직접 입력하여 제어합니다.

📂 디렉토리 및 파일 관리

pwd: 현재 작업 중인 폴더 경로 확인

ls -al: 현재 폴더의 파일 목록 보기 (숨김 파일 포함)

cd [폴더명]: 해당 폴더로 이동

mkdir [폴더명]: 새 폴더 생성

touch [파일명]: 빈 파일 생성

rm [파일명]: 파일 삭제 (rm -rf는 폴더 삭제)

📝 파일 내용 확인 및 편집

cat [파일명]: 파일 내용 전체 출력

head -n [숫자]: 파일의 앞부분 일부 출력

echo "내용" > [파일명]: 파일에 텍스트 저장

⚙️ 시스템 및 권한

sudo: 관리자(Root) 권한으로 명령 실행

ps aux: 실행 중인 모든 프로세스 확인

kill [PID]: 특정 프로세스 종료

chmod: 파일/폴더의 권한(읽기, 쓰기, 실행) 변경

6. 권한 구조 (chmod)

리눅스 파일 권한은 **소유자(U), 그룹(G), 기타 사용자(O)**로 나뉩니다.

r (4): 읽기

w (2): 쓰기

x (1): 실행

예: chmod 755 [파일명]은 소유자에게 모든 권한(4+2+1=7)을, 나머지에겐 읽기/실행 권한(4+1=5)을 부여합니다.
