# 🛠️ 데이터 엔지니어링 환경 구축 및 기술 로드맵

이 문서는 데이터 엔지니어가 되기 위한 기술 스택과 효율적인 개발 환경 구성을 위한 가상화 기술(Docker, VM)을 다룹니다.

## 1. 데이터 엔지니어 기술 스택 로드맵

데이터 엔지니어는 데이터를 수집, 변환, 저장하는 전체 시스템을 설계합니다.

| 분류 | 주요 기술 스택 | 상세 설명 |
| --- | --- | --- |
| **언어** | Python, Scala, Java | 데이터 처리 로직 및 자동화 스크립트 작성 |
| **데이터베이스** | MySQL, PostgreSQL, MongoDB | 관계형(SQL) 및 비관계형(NoSQL) 데이터 저장 |
| **빅데이터** | Hadoop, Spark, Hive | 대규모 분산 데이터 처리 및 쿼리 실행 |
| **워크플로우** | Airflow, Prefect | 복잡한 데이터 파이프라인의 스케줄링 및 관리 |
| **클라우드** | AWS, GCP, Azure | 인프라를 직접 구축하지 않고 빌려 쓰는 환경 |

## 2. 가상화 기술 (Virtualization)

물리적인 컴퓨터 한 대 위에 여러 개의 독립된 운영체제 환경을 만드는 기술입니다.

### 🔹 VM (Virtual Machine, 가상 머신)

- 하드웨어를 가상화하여 그 위에 새로운 운영체제(Guest OS)를 설치합니다.
- **장점:** 독립성이 매우 강함.
- **단점:** 리소스를 많이 차지하고 구동 속도가 무거움. (예: VMware, VirtualBox)

### 🔹 Docker (도커) - 컨테이너 기술

- 운영체제 전체를 가상화하는 대신, **프로세스 단위로 격리**하여 실행합니다.
- **특징:** 가볍고 빠르며, 어디서든 동일한 환경을 복제할 수 있습니다.
- **이미지(Image):** 실행에 필요한 모든 것이 담긴 템플릿.
- **컨테이너(Container):** 이미지를 실행한 상태.

## 3. WSL2와 개발 환경 최적화

윈도우 사용자가 리눅스 기반의 데이터 도구들을 원활하게 사용하기 위한 필수 설정입니다.

- **WSL2 선택 이유:** 실제 리눅스 커널을 사용하므로 파일 처리 속도가 훨씬 빠르고 **Docker Desktop**과의 호환성이 완벽합니다.
- **VS Code 연동:** 'Remote - WSL' 확장을 사용하면 윈도우용 VS Code에서 리눅스 내부 파일을 직접 편집할 수 있어 매우 편리합니다.

## 4. 데이터 파이프라인 구성의 실제 (실습 준비)

영상의 후반부에서 강조된 실무적인 준비 단계입니다.

1. **Ubuntu 설치:** WSL2를 통해 가장 대중적인 리눅스 배포판인 Ubuntu를 설치합니다.
2. **패키지 관리 (apt):** `sudo apt update` 명령어를 통해 최신 상태를 유지하는 법을 익힙니다.
3. **Python 가상환경 (venv):** 프로젝트마다 라이브러리 버전이 충돌하지 않도록 독립적인 공간을 만드는 것이 중요합니다.

## 5. 핵심 리눅스 명령어 (심화)

지난번 기초에 이어 파이프라인 관리 시 자주 사용하는 명령어입니다.

- `grep`: 파일 내에서 특정 단어를 찾을 때 사용 (예: `cat logs.txt | grep "Error"`)
- `find`: 특정 조건의 파일을 검색할 때 사용
- `tar -zxvf`: 대규모 데이터 셋이나 압축 파일을 풀 때 사용
- `curl` / `wget`: 인터넷에서 데이터를 직접 다운로드할 때 사용
